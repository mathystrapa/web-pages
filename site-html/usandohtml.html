O que é HTML?

HTML, sigla para "HyperText Markup Language", é uma linguagem de marcação utilizada para criar e estruturar páginas na web. Ele fornece a estrutura básica e os elementos necessários para definir o conteúdo de um site, permitindo a formatação, organização e exibição de informações na internet.

Origem e História

O HTML foi criado por Tim Berners-Lee, um cientista da computação britânico, em 1989 enquanto trabalhava na CERN (Organização Europeia para Pesquisa Nuclear). Naquela época, Berners-Lee estava procurando uma maneira de compartilhar informações e documentos científicos de forma eficiente entre seus colegas. Ele percebeu que, para atingir esse objetivo, precisaria de um sistema que permitisse a conexão entre diferentes documentos através de hiperlinks.

Em 1990, Berners-Lee publicou o primeiro rascunho do que viria a ser conhecido como HTML. A linguagem originalmente tinha apenas 18 elementos (tags), e as primeiras páginas da web eram bastante simples, com foco na estruturação básica do conteúdo.

Finalidade do HTML

A principal finalidade do HTML é fornecer uma estrutura básica para a criação de documentos que possam ser visualizados em um navegador web. Ele é a base de toda a web moderna e é responsável por permitir que os sites tenham conteúdos interativos e conectados através de links.

O HTML é uma linguagem de marcação, o que significa que ela não é uma linguagem de programação tradicional como C++ ou Python, mas sim uma linguagem que descreve a estrutura e organização dos elementos presentes em uma página web. Através do HTML, é possível criar parágrafos, títulos, imagens, formulários, tabelas e muito mais.

Sobre o HTML

O HTML é uma linguagem que utiliza "tags" para marcar elementos dentro de um documento. As tags são representadas por um conjunto de caracteres especiais entre colchetes angulares (<>). Cada tag tem um propósito específico e determina como o conteúdo dentro dela será estruturado ou formatado.

Além disso, o HTML pode ser combinado com outras tecnologias, como folhas de estilo CSS (Cascading Style Sheets) para estilizar o conteúdo e tornar as páginas mais atraentes visualmente, e JavaScript para adicionar interatividade e dinamismo aos elementos da página.

Com o HTML, os desenvolvedores podem criar sites estáticos, mas, em conjunto com outras tecnologias, é possível construir sites e aplicativos web complexos e interativos.

Estrutura do HTML

A estrutura do HTML é fundamental para a criação de páginas web e consiste em um conjunto de elementos que formam a base dos documentos HTML. Cada elemento é representado por uma tag, que é delimitada por colchetes angulares (<>). Essas tags definem a função e a estrutura dos diferentes elementos na página. A seguir, vamos explorar os principais componentes da estrutura do HTML:

Elementos Básicos

O elemento básico de um documento HTML é a tag <html>. Ela envolve todo o conteúdo da página e informa ao navegador que o arquivo é um documento HTML. Dentro da tag <html>, encontramos duas seções principais: <head> e <body>.

  • <head>: A seção <head> contém informações sobre o documento, como metadados, títulos e referências a arquivos externos, como folhas de estilo CSS ou scripts JavaScript. Essas informações não são exibidas diretamente na página, mas são essenciais para o funcionamento correto do documento.

  • <body>: A seção <body> contém o conteúdo principal da página, como textos, imagens, links, formulários e outros elementos visíveis que serão exibidos no navegador.

Comandos HTML

Mostrarei aqui uma lista com as principais tags de html. Estão separadas em tags de texto, organização e mídia.

  1. Tags textuais

    Em HTML (Hypertext Markup Language), as tags de texto são usadas para estruturar e formatar o conteúdo textual de uma página da web. Abaixo estão algumas das principais tags de texto em HTML, juntamente com uma breve explicação de suas funções:

    • <p></p>: Usada para delimitar parágrafos;
    • <h'n'></h'n'>: Usada para criar títulos, onde n pode ser 1, 2, 3, 4, 5 ou 6 de forma que 1 representa a maior fonte e 6 a menor fonte;
    • <strong></strong> ou <b><b>: Usada para deixar um texto em negrito;
    • <em></em> ou <i></i>: Usada para deixar um texto em itálico;
    • <u></u>: Deixa o texto sublinhado;
    • <s></s> ou <del></del>: Deixa o texto tachado;
    • <ins></ins>: Usada para indicar um texto inserido posteriormente ou de outro documento. Visualmente, não tem diferença da tag <u></u>;
    • <sub></sub>: Deixa um texto em forma de índice. Como essetextinho aqui em baixo;
    • <sup></sup>: Deixa um texto em forma de expoente. Como essetextinho aqui em cima;
    • <blockquote></blockquote>:

    • <abbr></abbr>: Cria uma abreviatura para um texto. Passe o mouse sobre esse texto para visualizar.;
    • <cite></cite>: Usada para indicar citação. Visualmente, não possui diferença das tags <em></em> ou <i></i>;
    • <code></code>: Usada para indicar um código. Exemplo: print('Hello, world!');

  2. Tags de organização

    As tags de organização são usadas para estruturar e organizar o conteúdo de uma página da web. Abaixo estão todas as principais tags de organização do HTML5:

    • <head></head>: Delimita o seu cabeçalho, onde serão colocadas as informações referente ao seu código html, como título, língua e outros;
    • <body></body>: Delimita o corpo do seu código html. É aqui onde são colocadas as informações que aparecerão ao usuário;
    • <header></header>: Usada para indicar o cabeçalho da página, onde geralmente existe uma logotipo, menus de navegação, etc.;
    • <nav></nav>: Usada em locais específicos da página com links de navegação, como um índice por exemplo;
      • Observação: Vale ressaltar a diferença de um sumário (ou índice) para um menu de navegação, pois este serve para facilitar a navegação entre páginas diferentes de um site e geralmente fica entre a tag <header></header>, enquanto aquele serve para facilitar a navegação entre tópicos da mesma página e geralmente fica entre a tag <nav></nav>.
    • <br>: Quebra de linha;
    • <hr>: Cria uma linha na horizontal;
    • Listas:
      • <ul></ul>: Lista sem ordem (aparece apenas um marcador);
      • <ol></ol>: Lista ordenada (aparecem números).
      Ambos os tipos de listas utilizam <li></li> para criar um item da lista;
    • <div></div>: Usada para organizar seu código. Não faz nenhuma diferença visual na página;
    • <!--'comentário'-->: Usada para criar um comentário no código. Não aparece na página;
    • <main></main>: Usada para indicar a parte principal da página. É recomendado que tenha apenas uma tag <main>;
    • <article></article>: Usada para agrupar um conteúdo independente;
    • <section></section>: Possui uma função semelhante à tag <article></article>, porém mais abraangente. É usada para agrupar conteúdos de mesmo valor semântico, mas não necessariamente independentes;
    • <aside></aside>: Usada para organizar conteúdos que não fazem parte da estrutura principal, como barras laterais, rodapés, etc;
    • <hgroup></hgroup>: Usada para agrupar vários cabeçalhos;
    • <figure></figure>: Usada para agrupar elementos de mídia, como imagens, vídeos ou áudios. A tag <figcaption></figcaption> dentro dela indica a legenda da imagem;
    • <footer></footer>: Usada para agrupar os elementos do rodapé.

  3. Tags de mídia

    • <img>: Usada para incorporar imagens a uma página web. Não precisa de fechamendo, o que siginfica que o conteúdo alternativo deve ser adicionado com o atributo alt='conteúdo_aternativo', que aparecerá caso a imagem não seja suportada pelo navegador do usuário. O atributo alt também ajuda os mecanismos de pesquisa a saberem exatamente sobre o que aquela imagem se trata. Outros atributos da tag <img> são: src = 'fonte_da_imagem' (pode ser um link https de onde a imagem está hospedada ou o caminho do arquivo em sua máquina); height = 'altura_em_pixels'; width = 'largura_em_pixels'; entre muitos outros atributos que podem ser incorporados ao CSS para estilizar a página web.

      Observação: Um gif também é considerado uma imagem, então basta adicionar o caminho do gif dentro do atributo src da tag <img>

    • <audio></audio>: Usada para inserir áudio, suportando diversos formatos de áudios diferentes. Atributos:

      1. 'src' = 'caminho_do_audio'(obrigatório): Indica o caminho do áudio, pode ser o endereço https onde está hospedado ou o caminho para o arquivo em sua máquina; É recomendado utilizá-lo dentro da tag <source> para, usando o atributo type, possamos especificar o formato do nosso arquivo.
      2. 'controls': Quando inserido, exibe opções de controle ao áudio, como play, pause, volume, etc. Sem ele, não aparecerá nada para o usuário e será necessário usar JavaScript para configurar botões de controle;

      3. 'preload' = 'auto, metadata ou none': Usado para indicar como o áudio deve ser pré-carregado pelo navegador. Existem três valores possíveis para o atributo:

        1. auto: O navegador decide automaticamente. Geralmente é a configuração padão;
        2. metadata: Apenas os metadados do áudio (duração, autor, título, etc) são pré-carregados;
        3. none: Nenhum metadado é pré-carregado. O áudio será baixado somente quando o usuário apertar o play.

      Em resumo, os atributos controls src e preload, juntamente com o elemento <source>, permitem personalizar a reprodução de áudio em uma página da web e garantir a compatibilidade com diferentes navegadores e formatos de áudio. O uso correto desses atributos e elementos ajuda a criar uma experiência de áudio mais acessível e funcional para os usuários.     • <track>: Usada para adicionar trilhas de texto a elementos de mídia, como áudio ou vídeo. As trilhas de texto geralmente são usadas para fornecer legendas, descrições de áudio ou outros tipos de informações textuais relacionadas ao conteúdo de mídia. A inclusão de trilhas de texto em suas mídias é uma prática importante para torná-las mais acessíveis a pessoas com deficiência auditiva ou para oferecer suporte a diferentes idiomas e contextos culturais. Aqui estão os atributos da tag <track>:

      1. src: Informa o caminho do arquivo .vtt que será a fonte da nossa legenda ou descrição de áudio. Um arquivo .vtt nada mais é do que um protocolo de formatação de texto para legendas e descrições de áudio. Você pode obter mais informações clicando aqui;
      2. kind: O atributo kind é obrigatório e é usado para especificar o tipo de trilha de texto. Os principais valores possíveis são:
        • 'subtitles': Esse valor é usado quando a trilha de texto contém legendas ou legendas para o conteúdo de mídia. As legendas são usadas principalmente para fornecer uma transcrição do diálogo ou do áudio em um idioma diferente do idioma principal do conteúdo de mídia. As legendas podem ser exibidas em vários idiomas para melhorar a acessibilidade e a compreensão do conteúdo;
        • captions: Esse valor é usado quando a trilha de texto contém legendas ocultas ou legendas que podem ser ativadas ou desativadas pelo usuário. As legendas ocultas são geralmente usadas quando o conteúdo de mídia possui áudio ou som, mas a trilha de texto oferece uma transcrição adicional, descrição de áudio ou legendas para acessibilidade. Os usuários podem ativar ou desativar essas legendas conforme necessário;
        • descriptions: Esse valor é usado quando a trilha de texto contém descrições de áudio para o conteúdo de vídeo. As descrições de áudio são usadas principalmente para fornecer informações sobre o que está acontecendo na tela para pessoas com deficiência visual. Por exemplo, uma descrição de áudio pode explicar as ações visuais, configurações ou personagens em um vídeo;
        • chapters: Esse valor é usado quando a trilha de texto contém informações sobre capítulos ou seções em um conteúdo de mídia. No entanto, o suporte para esse valor varia entre os navegadores, e nem todos os navegadores podem exibir capítulos ou seções definidos por esse tipo de trilha.
      3. srclang: Usado para fornecer o código do idioma da legenda;
      4. label: Rótulo ou descrição legível para que os usuários saibam qual o idioma da trilha de texto;
      5. default: Especifica qual das trilhas de texto (se houver mais de uma) é a padrão. Se nenhuma <track> possuir o atributo default, o navegador do usuário escolherá aquele com o código do idioma (atributo srclang) de acordo com suas configurações. Entretanto, o usuário pode mudar para o idioma desejado.
